# Markdown 写作

本节介绍用 Markdown 编写文档正文的常用语法，以及如何与 Docsify 配合（路由、锚点等）。

## 基础语法

### 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
```

标题会出现在侧栏的「当前页目录」中（由 `subMaxLevel` 控制层级），并自动生成锚点，便于链接到小节。

### 列表

无序列表：

```markdown
- 项目一
- 项目二
  - 子项
```

有序列表：

```markdown
1. 第一步
2. 第二步
```

### 链接

```markdown
[显示文字](链接地址)
```

站内跳转示例：

- 同目录：`[指南](guide.md)`
- 子目录：`[配置](config/options.md)`
- 锚点：`[核心概念](02-docsify.md#核心概念)`

### 图片

```markdown
![图片说明](图片地址)
```

图片可放在项目内（如 `assets/logo.png`）或使用外链。

### 代码块

行内代码：用反引号 `` `代码` ``。

块级代码：用三个反引号并注明语言（便于高亮）：

````markdown
```javascript
console.log('Hello');
```
````

Docsify 使用 Prism 做代码高亮，引入 Prism 主题后即可生效。

### 表格

```markdown
| 列一 | 列二 |
|------|------|
| A    | B    |
| C    | D    |
```

## 与 Docsify 的配合

### 文件名即路由

- 根目录下的 `README.md` 对应站点首页（`/#/`）。
- 其他 `xxx.md` 对应 `/#/xxx`。
- 子目录：如 `guide/install.md` 对应 `/#/guide/install`。若使用子目录，建议在该目录下单独配置 `_sidebar.md` 并开启 `relativePath: true`。

### 锚点

标题会自动生成锚点。链接到当前页某小节：`#锚点`；链接到其他页某小节：`其他页.md#锚点`。锚点格式因主题而异，一般为标题转成小写、空格变连字符等。

### 相对路径

站内资源（图片、其他 Markdown）建议用相对路径，这样在本地和 GitHub Pages 上都能正确打开。

## 可选扩展

- **Emoji**：Docsify 支持部分 emoji 简写（如 `:smile:`）。若不需要，可在配置中关闭。
- **公式**：需要数学公式时可引入 KaTeX 等插件，见官方文档。
- **流程图、时序图**：可集成 Mermaid 等插件。
